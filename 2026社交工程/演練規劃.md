# 2026 社交工程演練 — 規劃

> 本資料夾為 2026 年新一輪演練的開發目錄。基礎建設沿用 2025 版（追蹤後台 + LIFF 身分擷取 + MongoDB），
> 宣導頁改寫為「群組移轉」內部情境，**不沿用 2025 的振興加倍金（假冒政府）版本**。

## 壹、依據

113/12/20 資訊安全會議決議，執行公司內部「防範惡意連結透過 LINE 社交軟體發送」之工程演練服務計畫。

## 貳、演練內容

### 一、測試方式

- **測試對象**：具有加入公司 LINE 群的全體同仁（共 65 名）
- **發送方式**：管理部以「非講師」身份，發送一則含「新群組連結」的 LINE 訊息給群組全體同仁，
  統計受訊息引誘而點擊加入連結的同仁。
- **演練時間**：115/7/02

### 二、LINE 訊息主旨（誘餌文案）

```
各位同仁好，因本堂 AI-first 工作坊的階段性課程/實作已告一段落，為了配合管理部與經企室
後續的成果備份與 EIP 知識庫建立，這個臨時的「教學討論群」即將關閉。
後續的進階優化追蹤、工具更新與問題回覆，將全面移轉至公司核可的官方永久專屬社群。
本群組預計於本週五正式停用並解散，請各位同仁現在順手點擊下方連結，移轉至新成立的
官方專屬群組，以免漏掉後續的通知：
👉 [測試用新群組連結]
感謝大家的配合！
```

## 參、技術架構（沿用 2025 基礎）

| 元件     | 檔案                | 說明                                                                     |
| -------- | ------------------- | ------------------------------------------------------------------------ |
| 宣導頁   | `public/index.html` | 點擊後的資安教育揭露頁，已改寫為「群組移轉」情境                         |
| 追蹤後台 | `public/admin.html` | 統計總點擊數、不重複用戶、逐筆名單、匯出 CSV                             |
| 後端 API | `server.js`         | Express + MongoDB；LIFF 擷取點擊者 LINE 身分（userId / 顯示名稱 / 頭像） |
| 部署     | `vercel.json`       | Vercel；MongoDB 連線見 `MONGODB_SETUP.md`                                |

- 連結 → LIFF 頁（`index.html`）→ 取得 LINE 身分 → `POST /api/track` → 後台即時統計。
- 點擊者的 LINE 顯示名稱即可對應到同仁，供後續個別資安教育。

## 肆、注意事項

1. **演練後 1 小時內務必發出說明公告**（範本見 2025社交功能/ 內文件）。
2. 本次採內部「群組移轉」情境，**未假冒任何政府或外部單位**，避免冒用公務機關名義之法律風險。
3. 上線前用真實 LINE 帳號點一次，確認後台有跳出記錄（需先設好 Vercel 的 `MONGODB_URI`）。
